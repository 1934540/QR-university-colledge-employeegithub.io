# BolashaqQR

BolashaqQR - демонстрационная система учета посещаемости сотрудников университета через QR-коды. Проект создан для показа сценариев входа/выхода сотрудников, контроля опозданий, личного кабинета сотрудника и админ-панели без отдельного backend-сервера.

## Возможности

- Терминал входа с постоянным QR-кодом университета.
- Личный кабинет сотрудника в формате мобильного интерфейса.
- Сканирование QR-кода через камеру или симуляция сканирования для демонстрации.
- Проверка геолокации перед сканированием: QR можно использовать только в заданном радиусе от точки входа.
- Автоматическая фиксация прихода и ухода сотрудника.
- Расчет отработанного времени за день.
- Определение статусов посещения: вовремя, обычное опоздание, критическое опоздание, VIP, оправдано.
- Учет расписания преподавателей по дням недели и времени занятий.
- Отдельная логика для преподавателей и административного персонала.
- Настраиваемое системное время для тестирования разных сценариев.
- Админ-панель с PIN-входом.
- Дашборд со статистикой посещаемости за выбранный день.
- Список критических нарушителей.
- Управление базой сотрудников: добавление, редактирование, удаление.
- Управление расписанием преподавателей.
- Журнал посещаемости с поиском и фильтрами.
- Редактирование времени прихода и ухода.
- Оправдание опозданий с указанием причины и комментария.
- Экспорт журнала посещаемости в CSV.
- Хранение данных в `localStorage` браузера.
- Сброс данных к предустановленным демонстрационным значениям.

## Технологии

- Frontend: HTML5, CSS3, JavaScript
- UI: кастомные CSS-стили
- Иконки: Lucide Icons
- Графики: Chart.js
- QR-сканер: html5-qrcode
- Генерация QR-кодов: qrcodejs
- Хранение данных: Browser localStorage
- Локальный запуск: http-server через `npx`

## Установка

Склонируйте проект и перейдите в папку:

```bash
git clone https://github.com/username/bolashaq-qr.git
cd bolashaq-qr
```

Установка зависимостей не требуется, потому что проект использует CDN-библиотеки. Для удобного локального запуска нужен Node.js, чтобы выполнить скрипт `dev`.

## Запуск

```bash
npm run dev
```

После запуска откройте приложение в браузере:

```text
http://localhost:5173
```

Если `http-server` не установлен локально, команда автоматически запустит его через `npx`.

## Как пользоваться

### 1. Терминал входа

На главном экране отображается QR-код терминала входа. Он используется сотрудниками для фиксации прихода и ухода.

### 2. Кабинет сотрудника

Во вкладке кабинета можно выбрать сотрудника из списка, посмотреть профиль, расписание, историю отметок и выполнить сканирование.

Для демонстрации можно нажать кнопку симуляции сканирования без камеры.

Перед фиксацией прохода приложение проверяет геолокацию. Если сотрудник находится дальше разрешенного радиуса от точки входа, отметка не будет сохранена.

### 3. Админ-панель

Во вкладке администратора доступна панель управления.

PIN по умолчанию:

```text
1234
```

В админ-панели можно:

- смотреть статистику посещаемости;
- управлять сотрудниками;
- редактировать расписание преподавателей;
- просматривать журнал посещений;
- фильтровать записи;
- оправдывать опоздания;
- менять настройки системы;
- экспортировать данные в CSV.

## Логика учета опозданий

Для административного персонала проверяется стандартное начало рабочего дня. По умолчанию это `09:00`.

Для преподавателей учитывается расписание занятий:

- если преподаватель пришел до первой пары, статус считается нормальным;
- если пришел во время пары после ее начала, фиксируется критическое опоздание;
- если первая пара уже прошла, также фиксируется критическое опоздание;
- если пар в этот день нет, применяется стандартное начало рабочего дня.

VIP-сотрудники освобождены от контроля опозданий.

## Настройки

В админ-панели можно изменить:

- стандартное начало рабочего дня;
- количество обычных опозданий, которое превращается в одно критическое;
- порог попадания в список критических нарушителей;
- включение или отключение проверки геозоны;
- широту и долготу точки входа;
- допустимый радиус сканирования в метрах;
- PIN-код администратора.

Все настройки сохраняются в `localStorage`.

## Данные

Проект использует демонстрационные данные из файла `mockData.js`. При первом запуске они копируются в `localStorage`.

Если данные уже есть в браузере, приложение будет использовать сохраненную локальную версию.

Чтобы вернуться к исходным данным, используйте кнопку сброса в настройках админ-панели.

## Важное замечание

Проект является демонстрационным frontend-прототипом. PIN-код, сотрудники, журналы посещений и настройки хранятся на стороне браузера, поэтому текущая версия не предназначена для production-использования без backend, базы данных и полноценной авторизации.

Проверка геолокации работает через браузерный Geolocation API. Для реального использования страницу нужно открывать через HTTPS или `localhost`, а пользователь должен разрешить доступ к местоположению.
